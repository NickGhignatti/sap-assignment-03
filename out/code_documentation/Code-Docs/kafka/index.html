Group IDServizioTopic sottoscrittoRuolo
order-serviceordersaga-eventsAvanza la SAGA su ogni step event

drone-service-ordersdronedrone-requestsAssegna drone, avvia timer consegna

Consumer group semantics: tutte le repliche dello stesso servizio condividono un group.id. Kafka distribuisce le partizioni tra di loro automaticamente (load balancing). Servizi con group.id diverso ricevono ciascuno una copia indipendente dello stream (fan-out).

6. Infrastruttura — Note sui listener Kafka

Kafka gira in KRaft mode (nessun Zookeeper). Il broker espone due listener:

ListenerIndirizzoUsato da
PLAINTEXTkafka:9092Container Docker (comunicazione inter-servizio)
EXTERNALlocalhost:9094Macchina host (CLI Kafka, IDE, Kafka UI)

La variabile d’ambiente per i servizi è KAFKA_BROKERS=kafka:9092. Per accedere dall’host usare localhost:9094.
