DroneFlow — Architettura del Sistema

1. Panoramica

DroneFlow è una piattaforma a microservizi event-driven che gestisce l’intero ciclo di vita di una consegna con droni. Tre servizi indipendenti comunicano esclusivamente tramite messaggi AMQP su RabbitMQ; nessuna chiamata REST diretta esiste tra i backend.

6. Struttura del workspace Rust

workspace/
├── Cargo.toml              ← workspace root [members = [...]]
├── Cargo.lock
├── common/                 ← libreria condivisa (no binary)
│   └── src/
│       ├── lib.rs          ← pub mod events; pub mod messages;
│       ├── events.rs       ← SagaEvent, DroneEvent
│       └── messages.rs     ← OrderMessage
├── order/
│   └── src/
│       ├── main.rs         ← wiring
│       ├── api.rs          ← HTTP handlers (Axum)
│       ├── service.rs      ← OrderService + DTOs
│       ├── orchestrator.rs ← SagaOrchestrator
│       ├── saga.rs         ← SagaState, SagaStatus, SagaStep
│       ├── repository.rs   ← SagaRepository (MongoDB)
│       └── consumer.rs     ← AMQP consumer task
├── delivery/
│   └── src/
│       ├── main.rs
│       ├── service.rs      ← DeliveryService
│       └── consumer.rs     ← AMQP consumer task
└── drone/
    └── src/
        ├── main.rs
        ├── api.rs          ← HTTP handlers (Axum)
        ├── service.rs      ← DroneService
        ├── model.rs        ← DroneEntry, DroneState
        ├── store.rs        ← DroneEventStore (MongoDB)
        └── consumer.rs     ← AMQP consumer tasks (x2)
